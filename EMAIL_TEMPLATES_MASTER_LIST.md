### ğŸŒ Language Translation Requirements

All email templates must support **multi-language localization**:

#### Supported Languages
1. **English** (en) - Default/Primary
2. **Hebrew** (he) - Right-to-left (RTL) support required
3. **French** (fr) - Future expansion
4. Additional languages as needed

#### Implementation Approach

**Option 1: Template Variables (Recommended)**
```html
<!-- Subject -->
{{ emailTitle_en }} / {{ emailTitle_he }} / {{ emailTitle_fr }}

<!-- Content -->
<h1>{{ headerTitle }}</h1>
<p>{{ mainContent }}</p>
```

**Option 2: Separate Template Files**
```
welcome-email-en.html
welcome-email-he.html
welcome-email-fr.html
```

**Option 3: Backend i18n Integration**
```javascript
// Use backend translation service
const emailData = {
  locale: user.preferredLanguage, // 'en', 'he', 'fr'
  // Translations loaded from locale files
};
```

#### RTL Support for Hebrew
- Text direction: `dir="rtl"`
- Layout adjustments for right-aligned content
- Mirror horizontal spacing and alignment
- Social media icons order (right to left)

**Example RTL Wrapper:**
```html
<div dir="rtl" lang="he">
  <!-- Hebrew content here -->
</div>
```

#### Translation Workflow
1. **Create English template** (source)
2. **Extract translatable strings** to locale files
3. **Professional translation** (not machine translation)
4. **Test RTL layout** for Hebrew
5. **QA review** by native speakers
6. **Update documentation** with translation keys
